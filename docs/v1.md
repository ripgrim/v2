## 1. What was built

### The monorepo (spec §3, enforced by `scripts/check-boundaries.ts` in CI)

| Package | What it is |
|---|---|
| `@tripwire/contracts` | The shared language. Zod schemas + inferred types: demo-extracted domains (moderation, rules, runs-log, contributor, repo, insights, repo-content) + authored backend contracts (events, runs/Verdict, RuleResult, review muzzle, check, workflow DAG, RULE_CATALOG, DEFAULT_WORKFLOW). Imports nothing but zod. |
| `@tripwire/forge` | The seam: `ForgeAdapter` interface + types only (RawForgeEvent, DiffFile, ForgeAction incl. set-check, ContributorProfile). Nothing else, ever. |
| `@tripwire/core` | The pure engine. `defineRule`/`evaluateRule` (skipped-not-thrown envelope law), registry keyed `id@version`, 9 rules at `@1` (account-age, min-merged-prs, pr-rate-limit w/ spray CoV evidence, max-files-changed, english-only, crypto-address, honeypot, profile-readme, ai-review), workflow validate + executor (topo walk, gates, pause/resume), scoring taxonomy (0–100, red flags only subtract). No I/O — effects injected via RuleContext. |
| `@tripwire/forge-github` | The GitHub adapter: HMAC verify, payload→NormalizedEvent normalize, App JWT + installation token cache, GithubHttp/reads (diff/commits/file/contributor — no octokit anywhere), actions (comment upsert, `tripwire` check run, request-changes review, labels), captured fixture corpus + PROVENANCE.md. |
| `@tripwire/db` | Drizzle schema (13 tables: events, runs/run_steps/run_actions, repos/rule_configs/workflow_definitions, moderation_items, rollups_daily, better-auth tables + forge_identities) + the services layer (events tx-enqueue, runs snapshot persistence, repos/config CRUD, moderation decide, insights/rollups) + pg-boss queues + docker-CLI test-postgres helper. |
| `@tripwire/auth` | (owner-added) `./server`: createAuth + fail-closed posture guard. `./client`: browser auth client. |
| `@tripwire/utils` | generateId (UUIDv7, Bun fast path + portable fallback), errors, sleep, truncate, backoffWithJitter. |
| `@tripwire/ui` | Still empty — demo primitives live in `apps/web/src/components/ui`; lifting them is deliberate later work. |

### The three heads

**`apps/api`** (Hono, :8787)
- `POST /webhooks/github` — verify HMAC → ONE tx (insert raw event + pg-boss enqueue) → 200. UNIQUE(delivery_id) makes redelivery a no-op. Nothing else in the request path.
- `GET /events/stream` — SSE off Postgres LISTEN/NOTIFY, one dedicated client per connection, 15s heartbeats.
- `GET /healthz`.

**`apps/worker`** (pg-boss consumers)
- `process-event` — parse raw with contracts (failure ⇒ quarantine + fixture candidate), write NormalizedEvent + NOTIFY, pending check on change requests, match workflows (stored per-repo else DEFAULT_WORKFLOW), build RuleContext via adapter reads (per-read degradation), executor walk, persist run + steps with the workflow SNAPSHOT, record action rows first → execute → mark executed, emit comment + check from the same step.
- `resume-run` — moderation decisions walk the approve/deny edge off the snapshot; degraded runs approve⇒pass / deny⇒block.
- `rollup` — daily per-repo stats (cron 02:10, recomputes yesterday+today).
- ai generate — bounded tool loop via AI SDK + OpenRouter (`submit_review` tool carries the muzzle schema, step cap, diff up front, adapter-read tools only).

**`apps/web`** (TanStack Start, :3000) — routes:

| Route | What's in it |
|---|---|
| `/` | The demo's moderation queue (mock list — its rich shape outlives real data) with REAL stat cards from insights (pending/resolved/blocked-24h; bannedUsers honest zeros). |
| `/login` | GitHub sign-in via Better Auth. Gate lives in `__root.tsx` beforeLoad: open in dev without env, fail-closed in production. |
| `/events` | Live feed: server fn (cursor list) + EventSource → Query-cache merge. First end-to-end surface. Query errors render on-page. |
| `/runs/$runId` | The auditable run page: verdict/status chips, per-node step cards with rule evidence JSON, ai-review findings renderer, actions list. Every PR comment/check deep-links here. |
| `/rules` | Per-repo rule config CRUD from RULE_CATALOG: enabled toggle + zod-validated JSON config, repo picker. |
| `/workflows` | React Flow editor over the workflow DAG: node palette (rules/gates/actions), edge conditions, schema-checked save. Emits the exact JSON the executor eats (round-trip proven in tests). |
| `/moderation` | Paused runs awaiting decision; approve/deny → tx decide + resume job. |
| `/automod`, `/analytics`, `/$org/*`, `/profile/$userHandle` | Demo surfaces, still mock-backed by design — mocks shrink as later sessions give real data their depth. |
| `/api/auth/*` | Better Auth handler served BY the web head via `src/start.ts` request middleware (same-origin cookies, OAuth callback on :3000). |

### Security posture (hardening session, pre-live)
- **Fail-closed floor**: all-skipped or ≥50%-skipped rule nodes ⇒ needs_review + moderation item + degradation evidence — read-starving the worker can't mint green checks.
- **Auth fails closed in prod**: missing BETTER_AUTH_SECRET refuses to serve when NODE_ENV=production.
- **Block = failing check (primary) + request-changes review (friction on unprotected repos, best-effort — 403 on own PRs is legal).**
- **ai-review prompt** ported from the eve demo (maintainer-QoL test, slop signals, CONTRIBUTING check, ambiguity⇒needs_review) + hardened: trust rules (submission content is untrusted data; injection attempts are findings), explicit diff-truncation marker, AI-assistance product line, confidence anchors. Still `@1` — frozen at first live invocation.

### Test surface (116 tests, 4 CI checks green since commit one)
Per-rule units over fixture contexts · fast-check properties (score range,
red-flags-never-raise, determinism) · contract corpus parse/normalize ·
comment snapshots vs golden files · integration on REAL throwaway postgres
(tx+enqueue atomicity, duplicate delivery ⇒ one row, pipeline, moderation
pause/resume, PR-surface idempotency) · editor round-trip (emission →
validate → executor).

## 2. Caveats & issues we hit (and what each taught)

### Toolchain / runtime
1. **testcontainers hangs under Bun** — `start()` never resolves (dockerode stream plumbing), containers come up healthy. Replaced with a docker-CLI helper (`packages/db/src/testing.ts`), same real-postgres guarantee, faster (~1.6s suites). Later hardened with 3-attempt retry after parallel-suite races.
2. **`Bun is not defined`** — the web head's nitro dev runtime is **Node**, not Bun. `Bun.randomUUIDv7()` blew up the moment better-auth wrote its OAuth state row. generateId now has a portable RFC 9562 UUIDv7 fallback. Lesson: "Bun-locked stack" is true for api/worker, not for code that runs inside the web server runtime.
3. **ultracite 7.9.x broke against pinned Biome 2.4.5** — pinned ultracite to the demo's known-good 7.8.3.
4. **Bun.serve default 10s idleTimeout severed SSE** between our 15s heartbeats → `ERR_INCOMPLETE_CHUNKED_ENCODING` + reconnect loop. `idleTimeout: 45` on the api server.

### Env plumbing (the sneaky ones)
5. **Nitro's server-fn runtime does not inherit shell/bun env** — it loads dotenv from the app dir. Root `.env` was invisible to web server functions ("DATABASE_URL is not set" buried in a serialization frame). Convention now: root `.env` is truth, `apps/web/.env` is a symlink to it.
6. **`PORT` leaked into vite** through that symlink and moved the web server onto the api's port → renamed `API_PORT`.
7. **Raw multiline PEM broke Bun's .env parser** (could drop every var after it). The key must be quoted; worker also normalizes `\n` escapes now.
8. **Misnamed OAuth keys** — creds were added as `GITHUB_CLIENT_ID/SECRET`; the code reads `GITHUB_OAUTH_*` (deliberately distinct from the GitHub App creds). Renamed in place.
9. **`AI_REVIEW_MODEL` was documented but dead** — nothing read it. Now it's real: the worker's default model (OpenRouter slug); explicit rule config wins.
10. **Stale dev servers** — vite silently hops to :3001 when :3000 is taken; we debugged a "fixed" env against the old server for two rounds. Check `lsof -ti :3000` first.

### Framework fights (the auth transport saga)
11. **vite `server.proxy` never fires in this stack** — nitro owns the request pipeline. `/api/auth` fell through to the TanStack router, whose auth gate 307'd it to `/login`: the sign-in button was "doing nothing" because the client received login-page HTML. 
12. **This @tanstack/react-start version has no file-based server routes**, and **nitro's `server/routes` convention isn't picked up** by the nitro/vite beta either (with or without srcDir). What works: **`src/start.ts` → `createStart` request middleware**, serving `auth.handler(request)` before routing. Same-origin, no proxy, callback on :3000. (This is also why auth got its own package — server/client split became load-bearing.)
13. **TanStack server-fn serialization rejects `unknown`** — contracts grew a `jsonValueSchema`; rule configs are typed as JSON, which they are on the wire anyway.
14. **Duplicate drizzle instances** — adding drizzle-orm to web for one query created a second copy with incompatible private types; the query belonged in a db service anyway (`getRepoById`). The arrows were right.

### Product/spec judgment calls made under fire (all in DECISIONS.md)
15. **Fail-open → fail-closed** — step 6 shipped "skipped conducts as pass" (correct per-rule) which composed into "all reads fail ⇒ green check" (wrong system posture). Amended with the degradation floor.
16. **Demo vocabulary vs spec ontology** — contracts renamed to §4 names mid-run (`Rule`, `Run`, `RepoRef`…); web keeps deprecated aliases in shims that die with the mocks. Enum VALUES (flag/hide/close vs block/comment/label) still await reconciliation — deferred, tracked.
17. **The eve demo was missing, then found** (`~/tripwire-eve-agent-demo`) — ai-review prompts were authored fresh, then properly ported + hardened, all while still `@1` (no live runs exist, so the versioning law permits).
18. **Anthropic direct → OpenRouter** — one-file blast radius because §8's inversion kept the provider at the edge.
19. **Flaky-test forensics paid off** — a "flake" turned out to be a real precision bug (account-age fixture margin floored 2 days to 1 when the profile-fetch timestamp trailed the run clock).
